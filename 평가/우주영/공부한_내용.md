# 공부한_내용

## 1. WebRTC (Web Real-Time Communication)

---

WebRTC는 브라우저와 모바일 앱에서 **플러그인 없이** 실시간 음성, 영상, 데이터 통신을 가능하게 하는 오픈소스 기술.

### 핵심 특징

| 특징 | 설명 |
| --- | --- |
| P2P 통신 | 서버를 거치지 않고 브라우저 간 직접 연결하여 지연시간 최소화 |
| 내장 보안 | 모든 통신이 SRTP와 DTLS로 암호화 |
| NAT/방화벽 통과 | ICE, STUN, TURN 프로토콜로 복잡한 네트워크 환경에서도 연결 수립 |

### 주요 API

**MediaStream (getUserMedia)**

```jsx
const stream = await navigator.mediaDevices.getUserMedia({
    video: true,
    audio: true
});

```

**RTCPeerConnection**: P2P 연결 생성/관리, 코덱 협상, 암호화, 대역폭 관리

**RTCDataChannel**: 텍스트, 파일 등 임의의 데이터를 P2P로 전송 (TCP/UDP 지원)

### 연결 수립 과정 (시그널링)

1. Peer A가 **offer(SDP)** 생성 → 시그널링 서버로 전송
2. Peer B가 offer 수신 → **answer(SDP)** 생성하여 응답
3. 양측이 **ICE candidate** 교환하여 최적 경로 탐색
4. 직접 연결 수립 완료

### NAT 통과 메커니즘

| 서버 | 역할 |
| --- | --- |
| **STUN** | 클라이언트의 공인 IP와 포트를 알려줌 (대부분의 P2P 연결에 사용) |
| **TURN** | 직접 연결 불가 시 중계 역할 (트래픽 비용 발생, 연결 성공률 향상) |
| **ICE** | STUN과 TURN을 조합하여 가능한 모든 연결 경로 시도 |

### 활용 사례

- 화상회의 (Google Meet, Zoom Web)
- 라이브 스트리밍 (초저지연 1초 미만)
- P2P 파일 공유 (ShareDrop, WebTorrent)
- 온라인 협업 도구 (Figma 스타일)
- 클라우드 게이밍

## 2. WebSocket & STOMP

### WebSocket 개요

WebSocket은 클라이언트와 서버 간 **양방향 실시간 통신**을 가능하게 하는 프로토콜입니다. HTTP와 달리 한 번 연결되면 연결을 유지하면서 양쪽에서 자유롭게 데이터를 주고받을 수 있습니다.

### HTTP vs WebSocket 비교

| 항목 | HTTP | WebSocket |
| --- | --- | --- |
| 통신 방식 | 요청-응답 (단방향) | 양방향 |
| 연결 유지 | 매 요청마다 새 연결 | 연결 유지 (Persistent) |
| 오버헤드 | 매 요청마다 헤더 포함 | 최초 핸드셰이크 후 경량 프레임 |
| 실시간성 | 폴링 필요 | 즉시 푸시 가능 |
| 적합한 용도 | REST API, 정적 리소스 | 채팅, 게임, 실시간 알림 |

### STOMP (Simple Text Oriented Messaging Protocol)

STOMP는 WebSocket 위에서 동작하는 **메시징 프로토콜**입니다. WebSocket은 단순히 바이트 스트림만 제공하는데, STOMP를 사용하면 메시지 형식과 라우팅 규칙을 정의할 수 있습니다.

**STOMP의 장점:**

- **구독(Subscribe) 모델**: 특정 주제(Topic)를 구독하면 해당 주제로 발행된 메시지를 자동으로 받음
- **브로드캐스트 용이**: 방 전체에 메시지를 쉽게 전송
- **Spring 통합**: Spring WebSocket과 자연스럽게 연동

### STOMP 메시지 구조

```
COMMAND
header1:value1
header2:value2

Body^@

```

**주요 Command:**

- `CONNECT`: 서버에 연결
- `SUBSCRIBE`: 특정 destination 구독
- `SEND`: 메시지 전송
- `MESSAGE`: 서버에서 클라이언트로 메시지 전달

### Spring Boot + STOMP 구성 예시

```java
@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void configureMessageBroker(MessageBrokerRegistry config) {
        config.enableSimpleBroker("/topic", "/queue");  // 구독 경로
        config.setApplicationDestinationPrefixes("/app"); // 발행 경로
    }

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws").withSockJS();  // 연결 엔드포인트
    }
}

```

### 프로젝트에서의 활용

| 기능 | destination 예시 |
| --- | --- |
| 게임방 채팅 | `/topic/room/{roomId}/chat` |
| 게임 상태 동기화 | `/topic/room/{roomId}/state` |
| 채점 결과 알림 | `/topic/room/{roomId}/result` |
| 개인 알림 | `/queue/user/{userId}/notification` |

## 3. Redis

### 개요

Redis는 **인메모리 데이터 저장소**로, 캐시, 세션 저장, 메시지 브로커 등 다양한 용도로 사용. 모든 데이터를 메모리에 저장하기 때문에 매우 빠른 읽기/쓰기 성능 제공

### 주요 특징

| 특징 | 설명 |
| --- | --- |
| 인메모리 | 모든 데이터를 RAM에 저장하여 초고속 접근 |
| 다양한 자료구조 | String, List, Set, Hash, Sorted Set 등 지원 |
| 영속성 옵션 | RDB 스냅샷, AOF 로그로 데이터 보존 가능 |
| Pub/Sub | 발행-구독 메시징 패턴 지원 |
| 클러스터링 | 수평 확장 가능 |

### Redis 자료구조 활용 예시

```
String  → 단순 캐시, 세션 토큰
Hash    → 사용자 정보, 게임방 설정
List    → 메시지 큐, 최근 기록
Set     → 중복 없는 집합 (온라인 유저 목록)
Sorted Set → 랭킹, 점수 기반 정렬

```

### 프로젝트에서의 활용

**1. 세션/캐시 저장**

```
SET user:session:{userId} {sessionData} EX 3600

```

**2. 게임방 상태 관리**

```
HSET room:{roomId} status "PLAYING"
HSET room:{roomId} players "[user1, user2]"
HSET room:{roomId} timer 300

```

**3. 인원 제한 동시성 제어**

```
INCR room:{roomId}:count
// 최대 인원 초과 시 입장 거부

```

**4. 타이머 관리**

```
SET room:{roomId}:deadline {timestamp} EX 600

```

### Redis Pub/Sub

Pub/Sub은 **발행-구독 패턴**으로, 메시지를 발행하면 해당 채널을 구독한 모든 클라이언트가 메시지를 받습니다.

**활용 시나리오:**

- 채점 결과를 방 전체에 브로드캐스트
- 서버 인스턴스가 여러 개일 때 메시지 동기화
- 실시간 이벤트 전파

```
PUBLISH room:123:events '{"type":"SUBMIT_RESULT","userId":"user1","passed":true}'

```

### Spring Boot + Redis 연동

```java
@Configuration
public class RedisConfig {
    @Bean
    public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory factory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(new GenericJackson2JsonRedisSerializer());
        return template;
    }
}

```

## 4. 웹 푸시 & FCM (Firebase Cloud Messaging)

### 웹 푸시 개요

웹 푸시는 브라우저가 닫혀 있어도 **서버에서 클라이언트로 알림을 보낼 수 있는** 기술입니다. Service Worker를 통해 백그라운드에서 푸시 메시지를 수신합니다.

### 웹 푸시 동작 원리

```
1. 사용자가 알림 권한 허용
2. 브라우저가 Push Service에 구독 요청 → 구독 정보(endpoint, keys) 획득
3. 서버가 구독 정보를 저장
4. 서버가 Push Service에 메시지 전송 요청
5. Push Service가 해당 브라우저로 메시지 전달
6. Service Worker가 메시지 수신 → 알림 표시

```

### FCM (Firebase Cloud Messaging)

FCM은 Google에서 제공하는 **무료 푸시 알림 서비스**입니다. 웹, Android, iOS 모두 지원합니다.

**FCM의 장점:**

- 무료 (제한 없음)
- 크로스 플랫폼 지원
- 토픽 기반 브로드캐스트
- 안정적인 전송 보장

### FCM 구성 요소

| 구성 요소 | 역할 |
| --- | --- |
| FCM 서버 | 메시지 라우팅 및 전달 |
| 앱 서버 | 메시지 생성 및 FCM에 전송 요청 |
| 클라이언트 | 메시지 수신 및 표시 |
| 토큰 | 기기/브라우저 식별자 |

### 프론트엔드 구현 (Firebase SDK)

```jsx
// Firebase 초기화
import { initializeApp } from 'firebase/app';
import { getMessaging, getToken, onMessage } from 'firebase/messaging';

const app = initializeApp(firebaseConfig);
const messaging = getMessaging(app);

// 토큰 획득
const token = await getToken(messaging, { vapidKey: 'YOUR_VAPID_KEY' });
// 토큰을 서버에 저장

// 포그라운드 메시지 수신
onMessage(messaging, (payload) => {
    console.log('메시지 수신:', payload);
    // 알림 표시
});

```

### Service Worker (firebase-messaging-sw.js)

```jsx
importScripts('https://www.gstatic.com/firebasejs/9.0.0/firebase-app-compat.js');
importScripts('https://www.gstatic.com/firebasejs/9.0.0/firebase-messaging-compat.js');

firebase.initializeApp({
    // config
});

const messaging = firebase.messaging();

// 백그라운드 메시지 수신
messaging.onBackgroundMessage((payload) => {
    const notificationTitle = payload.notification.title;
    const notificationOptions = {
        body: payload.notification.body,
        icon: '/icon.png'
    };
    self.registration.showNotification(notificationTitle, notificationOptions);
});

```

### 백엔드 구현 (Spring Boot + Firebase Admin SDK)

```java
@Service
public class FCMService {

    public void sendMessage(String token, String title, String body) {
        Message message = Message.builder()
            .setToken(token)
            .setNotification(Notification.builder()
                .setTitle(title)
                .setBody(body)
                .build())
            .build();

        FirebaseMessaging.getInstance().send(message);
    }

    // 토픽 기반 브로드캐스트
    public void sendToTopic(String topic, String title, String body) {
        Message message = Message.builder()
            .setTopic(topic)
            .setNotification(Notification.builder()
                .setTitle(title)
                .setBody(body)
                .build())
            .build();

        FirebaseMessaging.getInstance().send(message);
    }
}

```

## 실시간 통신 기술 비교

| 기술 | 용도 | 특징 |
| --- | --- | --- |
| **WebSocket + STOMP** | 실시간 양방향 통신 | 연결 유지, 즉시 전송 |
| **WebRTC** | P2P 음성/영상/데이터 | 서버 경유 없이 직접 연결 |
| **Redis Pub/Sub** | 서버 간 메시지 동기화 | 다중 서버 환경에서 브로드캐스트 |
| **FCM (웹 푸시)** | 오프라인 알림 | 브라우저 닫혀 있어도 전송 가능 |
